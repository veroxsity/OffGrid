# Yggdrasil Mesh Networking Basics (Linux)

> TL;DR: Install Yggdrasil from the official repo, generate a fresh config, enable and start the service, add a few public peers, then verify you have a global Yggdrasil IPv6 and can ping other nodes. Restrict any services you expose to the yggdrasil0 interface.

Yggdrasil creates an encrypted IPv6 mesh overlay between nodes.

## Install

```bash
curl -s https://repo.yggdrasil-network.github.io/install.sh | sudo bash
sudo apt install -y yggdrasil
```
What this does: Adds the upstream apt repository and installs the daemon and CLI.

## Configure

```bash
sudo yggdrasil -genconf | sudo tee /etc/yggdrasil.conf
sudo systemctl enable --now yggdrasil
```

Edit peers in `/etc/yggdrasil.conf` to add public peers for faster convergence.

## Verify

```bash
ip -6 addr show yggdrasil0 | sed -n '/scope global/p'
yggdrasilctl getPeers || sudo yggdrasilctl getPeers
ping6 -c 3 200::/7 || true
```
Expected: yggdrasil0 has a global IPv6 address; `getPeers` shows connected peers.

## Troubleshooting
- Service won’t start
  - Check config syntax: `sudo yggdrasil -useconffile /etc/yggdrasil.conf -normalise -dumpconf >/dev/null`
  - Logs: `journalctl -u yggdrasil -n 200 --no-pager`
- No peers/route
  - Add multiple public peers; ensure outbound UDP/TCP not blocked by firewall.
- Can’t reach other nodes
  - Verify your ISP/CGNAT doesn’t block needed egress; try different peers.

## Security hardening
- Bind any self-hosted services to yggdrasil0 only, not 0.0.0.0/::
- Use host firewall rules to restrict inbound to yggdrasil0 and trusted subnets
- Keep Yggdrasil updated; rotate keys if you suspect compromise

## Uninstall / cleanup
```bash
sudo systemctl disable --now yggdrasil
sudo apt -y purge yggdrasil
sudo rm -f /etc/yggdrasil.conf
```
What this does: Stops the mesh daemon, removes the package, and deletes its config.

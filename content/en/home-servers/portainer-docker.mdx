# Portainer (Docker UI) with Docker

Portainer lets you manage Docker visually.

> TL;DR: Deploy Portainer CE, set a strong admin password, add your local Docker endpoint, and back up `./data`.

## Deploy

```bash
mkdir -p ~/portainer && cd ~/portainer
cat > docker-compose.yaml <<'YAML'
services:
  portainer:
    image: portainer/portainer-ce:latest
    container_name: portainer
    ports:
      - "9443:9443"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - ./data:/data
    restart: unless-stopped
YAML

docker compose up -d
```

What this does: Launches Portainer with access to the local Docker socket and a persistent data dir.

## Verification
- Open `https://SERVER_IP:9443` → set admin password → add local environment

## Tips
- Use Stacks to deploy Compose YAMLs
- Enable backups under Settings and also back up `./data`

## Troubleshooting
- 502/Bad Gateway
  - Wait for first-time setup to complete; check container logs
- Permission errors
  - Ensure your user can access the Docker socket or rely on root-only access

## Security hardening checklist
- Restrict access to 9443 to trusted IPs; put behind reverse proxy + SSO
- Enable TLS and enforce strong admin credentials
- Consider agent mode for remote endpoints (don’t expose docker.sock widely)

## Uninstall / cleanup
```bash
docker compose down
rm -rf ./data
```
